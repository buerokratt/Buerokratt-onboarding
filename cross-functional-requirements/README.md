### Bürokratt cross-functional requirements for project management

- [ ] Every issue must be based on a business-level [user story](./project-management/user-story.md)
- [ ] Developments may not be started until [acceptance criteria](./project-management/acceptance-criteria.md) have been set
- [ ] All developments must be based on a list of accepted technical stack
- [ ] Requirements for code commits must be followed
- [ ] All developments must pass acceptance tests
- [ ] All developments must follow the rules for documentation
- [ ] Deployment
- [ ] Board statuses
